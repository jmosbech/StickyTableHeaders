Note that in order to make the table header look ok when scrolling we need to add a background color and a bottom border.

There is also a bug in IE7 that causes the leftmost column to resize when scrolling. Consider using conditional comments (see source) to make sure the plugin is not loaded in this browser.

When using bootstraps table-responsive there are two scroll container. The window for vertical scrolling and the table-responsive div for horizontal scrolling. Make sure you add the table-responsive div as additional scrollable area to the stickytableheaders call parameters.
